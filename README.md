# WEB Developer Test Task — Geo Orders Service

Продакшн-деплой: http://185.233.45.170:4173

Цей проєкт реалізовано в межах тестового завдання. Його мета — створення сервісу для імпорту, збереження та перегляду замовлень з географічною прив'язкою в межах штату New York із використанням просторових можливостей PostGIS.

Система дозволяє визначати округ доставки за координатами точки, розраховувати відповідні податкові ставки та забезпечує повний цикл обробки замовлення — від імпорту до збереження та відображення в адмін-панелі.

---

## Архітектура

Проєкт побудований на:

- Frontend: React + TypeScript
- Backend: Node.js + Express
- База даних: PostgreSQL + PostGIS
- Контейнеризація: Docker + Docker Compose

Просторові операції виконуються безпосередньо на рівні бази даних.

Для визначення округу використовується `ST_Contains`, а координати перетворюються в:

```sql
ST_SetSRID(ST_MakePoint(longitude, latitude), 4326)
```

Для оптимізації запитів додано GiST-індекси на геометричні поля.

---

## Геодані

Межі округів були імпортовані з TIGER/Line Shapefiles (US Census Bureau).
Shapefile імпортовано через `shp2pgsql` з подальшим створенням просторового індексу.

Джерела даних:

- Tax Rates: https://www.tax.ny.gov/pdf/publications/sales/pub718.pdf
- Counties (TIGER 2023): https://www2.census.gov/geo/tiger/TIGER2023/COUNTY/tl_2023_us_county.zip

## Функціонал

Система підтримує:

- Імпорт замовлень із CSV
- Валідацію координат
- Автоматичне визначення округу через просторовий запит
- Розрахунок податку
- Збереження геометрії точки
- Cursor-based пагінацію
- Сортування (ASC / DESC)
- Пошук за містом
- Ручне створення замовлення через UI
- Обробку помилок імпорту зі статистикою

Пагінація реалізована без OFFSET — використовується cursor-підхід, що забезпечує стабільність при зростанні об'єму даних.

---

## Авторизація

У проєкті реалізована JWT-авторизація з використанням HTTP-only cookies.

Після успішного логіну:

- сервер генерує JWT
- токен зберігається в HTTP-only cookie
- cookie автоматично передається з кожним запитом
- middleware перевіряє валідність токена
- доступ до захищених ендпоінтів дозволяється лише авторизованим користувачам

Така реалізація:

- захищає токен від XSS (через HTTP-only)
- не потребує збереження токена в localStorage
- дозволяє централізовано керувати сесією

## База даних

База розгортається в Docker-контейнері на основі образу PostGIS.
Під час старту завантажується підготовлений dump із вже імпортованими геоданими та налаштованими індексами.

Передбачена можливість повного бекапу та відновлення БД через `pg_dump`.

## Запуск проєкту

1. Clone repository

```bash
git clone https://github.com/Amigo20057/web_dev_test_task.git
```

2. Copy env files

```bash
cp .env.example .env
cp backend/.env.example backend/.env
cp client/.env.example client/.env
```

3. Start project

```bash
docker compose up --build
```

4. Open in browser: http://localhost:4173
